# Codegen Type Options: JAVA vs CPP

The `codegen.type` field in `_config.prop` controls which language skeleton FD Headless generates for your SeamOS application.

## JAVA

- **Produces**: Maven project with Java source skeleton under `<ProjectName>_<AppName>/`
- **Build tool**: Maven (`mvn.args` field is passed through to the Maven invocation)
- **When to use**:
  - Default recommendation for most SeamOS applications
  - When targeting SeamOS runtimes with JRE 21 (Eclipse Temurin 21 LTS)
  - When using the SeamOS Java App Framework APIs
  - When your team is more familiar with Java tooling

## CPP

- **Produces**: CMake-based C++ source skeleton
- **Build tool**: CMake / make (Maven args are not used)
- **When to use**:
  - Performance-critical applications with tight real-time constraints
  - Integrating with existing C/C++ libraries or legacy codebases
  - Targets with limited JVM support

## Config property

```properties
# JAVA skeleton (default)
codegen.type=JAVA

# CPP skeleton
codegen.type=CPP
```

## Platform notes

- Both types are generated by the same `seamos-fd-headless` Docker image
- The `--platform linux/amd64` flag is always applied regardless of codegen type
- Apple Silicon (arm64) users require Rosetta 2 for both types

## mvn.args (JAVA only)

For JAVA codegen, `mvn.args` is forwarded to Maven. Leave empty for default behavior:

```properties
mvn.args=
```

Example with proxy settings:

```properties
mvn.args=-Dhttp.proxyHost=proxy.corp -Dhttp.proxyPort=8080
```

`mvn.args` is ignored when `codegen.type=CPP`.
