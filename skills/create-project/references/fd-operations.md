# FD Headless Operations

Three operations supported by the FD Headless 8.6.0 Docker image. The `create-project` skill wraps only these three operations. Other operations are out of scope.

## GENERATE_FSP

Create a new FSP project (empty project + skeleton).

- **Input**: interface JSON (branch/config array), project name, `"Custom UI"` (fixed)
- **Output**: FSP project files (`.project`, `.settings/`, FSP artifacts, etc.) under `<workspace>/<project-name>/`
- **Precondition**: workspace directory is empty or does not exist (or use `--force-clean`)

## GENERATE_SDK_APP

Generate an SDK app skeleton from an existing FSP project.

- **Input**: interface JSON, existing FSP project path, project name, UI type
- **Output**: SDK app source tree (C/C++ or platform-specific)
- **Precondition**: target workspace already contains an FSP project generated by `GENERATE_FSP`

## UPDATE_SDK_APP

Regenerate (incrementally update) an existing SDK app after interface changes.

- **Input**: new interface JSON, existing SDK app path, project name, UI type
- **Output**: SDK app source updated to reflect the changed interface
- **Precondition**: workspace where `GENERATE_SDK_APP` has been run at least once

## Success / Failure Detection

All three operations are determined by **stdout strings** (not exit code):

- Success: `FD HEADLESS EXECUTION COMPLETED SUCCESSFULLY`
- Failure: `FD HEADLESS EXECUTION EXITED WITH ERRORS`

The skill orchestrator (`scripts/create-project.sh`) uses `grep -qF` to detect these two strings.
