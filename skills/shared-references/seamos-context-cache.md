# SeamOS Context Cache (`.seamos-context.json`)

## Overview

The `.seamos-context.json` file is a shared context cache used by multiple SeamOS skills to remember the last-used device and app across sessions. This reduces repetitive selection dialogs and improves user experience during iterative development and testing workflows.

## File Location

- **Path**: Workspace root (same directory as `seamos-assets/`, `skills/`, etc.)
- **Filename**: `.seamos-context.json`
- **Git status**: Not committed (should be in `.gitignore`)

## Cache File Structure

```json
{
  "deviceId": "42",
  "deviceName": "RV-C1000 (SN: SN-001)",
  "appId": "10250",
  "appName": "Test App",
  "updatedAt": "2026-04-02T10:30:00Z"
}
```

### Field Definitions

| Field | Type | Source | Mutable | Purpose |
|-------|------|--------|---------|---------|
| `deviceId` | string | SDM MCP (list_devices) | Yes | Unique device identifier |
| `deviceName` | string | SDM MCP (list_devices) | Yes | Human-readable device name with serial (e.g., "RV-C1000 (SN: SN-001)") |
| `appId` | string | SDM MCP (list_apps or list_installed_apps) | Yes | Unique app identifier |
| `appName` | string | SDM MCP (list_apps or list_installed_apps) | Yes | Human-readable app name |
| `updatedAt` | ISO 8601 string | Generated by skill | Yes | Timestamp of last cache update (e.g., "2026-04-02T10:30:00Z") |

## Shared Ownership

This cache is **shared across multiple skills**. Each skill manages different fields based on its responsibility:

### `manage-device-app` Skill
- **Responsibility**: Device and app management on physical devices
- **Read/Write**: ALL fields (`deviceId`, `deviceName`, `appId`, `appName`, `updatedAt`)
- **Behavior**: Overwrites the entire cache after any successful action (install, update, uninstall)
- **Note on uninstall**: Even after uninstalling an app, the cache is updated with the appId — the user may want to reinstall or manage the same app on a different device next time

### `update-app` Skill
- **Responsibility**: Uploading new versions to the marketplace
- **Read/Write**: App fields only (`appId`, `appName`, `updatedAt`)
- **Read-only**: Device fields (`deviceId`, `deviceName`) — **must be preserved** if already present
- **Behavior**: After a successful version upload (2xx response), reads the existing cache and updates only `appId`, `appName`, and `updatedAt`, while preserving `deviceId` and `deviceName`

## Usage Pattern

### Cache Check Before Selection

When a skill needs to select a device or app, it should:

1. **Read** the cache file (if it exists)
2. **Verify** the cached ID exists in the current MCP response (e.g., `list_devices` or `list_apps`)
3. **Check status** (for devices: online/offline; for apps: availability)
4. **Show confirmation** if the cache is valid:
   ```
   이전에 사용한 [디바이스/앱]: {name} (ID: {id}) — 이대로 진행할까요? (Y/다른 [디바이스/앱] 선택)
   ```
5. **Proceed with full list** if cache is invalid or user declines

### Cache Update After Success

When an action completes successfully:

1. **Read** the existing cache (if it exists) to preserve unmodified fields
2. **Update** only the fields owned by the current skill
3. **Write** the updated cache back using the Write tool

Example for `update-app`:
```json
{
  "deviceId": "{preserved from existing file, or omit if not present}",
  "deviceName": "{preserved from existing file, or omit if not present}",
  "appId": "{newly selected appId}",
  "appName": "{newly selected app name}",
  "updatedAt": "{current ISO 8601 timestamp}"
}
```

Example for `manage-device-app`:
```json
{
  "deviceId": "{newly selected deviceId}",
  "deviceName": "{newly selected device name}",
  "appId": "{newly selected appId}",
  "appName": "{newly selected app name}",
  "updatedAt": "{current ISO 8601 timestamp}"
}
```

## Implementation Guidelines

- **File I/O**: Use the Read and Write tools provided by the skill runtime
- **JSON validity**: Always generate valid JSON with proper escaping (especially in `deviceName` and `appName` strings)
- **Timestamps**: Use ISO 8601 format (e.g., `2026-04-02T10:30:00Z`)
- **Null/undefined fields**: Do not include fields with null or undefined values — omit them entirely if not applicable
- **Atomicity**: Write the complete cache in a single operation to avoid partial updates

## Important Notes

- **No schema enforcement**: This is a simple key-value store with no validation. Skills trust each other to write valid data.
- **Non-critical**: The cache is optional for skill operation. If it doesn't exist or is invalid, skills should proceed with interactive selection.
- **User cleanup**: Users can manually delete `.seamos-context.json` to reset cached selections at any time.

## create-project

`skills/create-project/` 스킬이 성공 종료 시 프로젝트 루트 `.seamos-context.json` 의 `last_project` 키를 atomic upsert (flock + `.tmp` + `mv`) 한다.

### Schema

```json
{
  "last_project": {
    "name": "<project-name>",
    "workspace_path": "<absolute-path>",
    "operation": "GENERATE_FSP" | "GENERATE_SDK_APP" | "UPDATE_SDK_APP",
    "image_tag": "public.ecr.aws/g0j5z0m9/seamos-fd-headless:<version>",
    "interface_json_sha256": "<64-hex>",
    "created_at": "<ISO-8601 UTC>"
  }
}
```

- `name` — `--project-name` 인자 값
- `workspace_path` — 생성된 FSP 프로젝트의 절대 경로 (스킬 실행 시 `--workspace` 값의 절대 경로)
- `operation` — 수행된 FD 동작 (3종 중 하나)
- `image_tag` — 실제 사용된 Docker 이미지 태그
- `interface_json_sha256` — `<workspace>/_interface.json` 의 SHA256 (재실행 시 변경 감지용)
- `created_at` — 성공 완료 시점 (UTC)

### 후속 스킬에서 읽기

후속 스킬(`build-fif`, `manage-device-app`, 기타 FD 체인) 은 `.seamos-context.json` 의 `last_project` 값을 자동 참조하여 사용자가 프로젝트 경로를 매번 지정하지 않아도 되게 한다.

```bash
# 최근 프로젝트 워크스페이스 읽기
LAST_WS=$(jq -r '.last_project.workspace_path' .seamos-context.json)

# 최근 프로젝트 이름
LAST_NAME=$(jq -r '.last_project.name' .seamos-context.json)

# interface JSON 해시 (변경 감지)
LAST_IFACE_SHA=$(jq -r '.last_project.interface_json_sha256' .seamos-context.json)
```

### Concurrency

`create-project.sh` 는 쓰기 시 `.seamos-context.json.lock` 으로 `flock` 상호배제 + `.tmp` 임시파일 후 `mv` 로 atomic 교체. 두 스킬 인보케이션이 동시에 실행되어도 JSON 파일은 항상 valid.
